Why Konva?

HTML hitboxes alone could not match the illustrated hero map. Konva lets the app draw polygons, control opacity, and trigger tweens for glow and hover states while keeping React state in sync.

State storage

All data lives in local arrays (workExperience.ts, education.ts, projects.ts). No API calls or database; deployable as a static bundle.

PortfolioCover → Konva flicker & hotspots

Custom cursor, polygon hit detection, and timed neon tweens.

▼

Hotspots are defined as point arrays. getBounds() derives text placement. Hover events call line.to() for instant glow changes.

Timed tweens run every few seconds for each polygon, creating the arcade-like flicker. Click handlers map IDs to navigation callbacks (work, education, contact, tools, projects).

// hover feedback
line.to({
  opacity: isHover ? 0.98 : 0.7,
  shadowBlur: isHover ? 28 : 12,
  duration: 0.08,
});

// per-hotspot flicker timer
setInterval(() => {
  new Konva.Tween({ node: line, opacity: 0.62 + Math.random()*0.12 }).play();
}, 2300);

ProjectsTabs → search & tag filter

URL-driven tab, memoized filters, and resilient defaults.

▼

Initial tab is read from ?tab=frontend|fullstack; effect keeps state synced if the URL changes.

useMemo builds a tag set and filters by free-text query (title, description, tags) plus tag selection, preventing extra re-renders.

const filtered = useMemo(() => {
  const ql = q.trim().toLowerCase();
  return data.filter(p => {
    const matchesQ = !ql ||
      p.title.toLowerCase().includes(ql) ||
      p.description.toLowerCase().includes(ql) ||
      p.tags?.some(t => t.toLowerCase().includes(ql));
    const matchesTag = !tag || p.tags?.includes(tag);
    return matchesQ && matchesTag;
  });
}, [q, tag, data]);

BackgroundTabs → timeline layout

Radix tabs over static arrays with square timeline primitives.

▼

Tabs swap between workExperience and education arrays. Timeline items use a shared vertical rail and square avatar initials derived from the company or institution.

Data-only structure keeps content editable without touching component logic.

function initials(name) {
  return name.split(/\s+/)
    .filter(Boolean)
    .slice(0,2)
    .map(part => part[0]?.toUpperCase() ?? "")
    .join("");
}

PaperFoldWar → local/online modes

Router-mounted game pages with shared engine for moves and state sync.

▼

/games/paper-fold-war loads the local engine; the online version mounts under /games/paper-fold-war/online/:roomId.

State for hands/turns is held in React; online mode adds room param handling and network sync logic (Supabase-style or socket-ready).

const { roomId } = useParams();
useEffect(() => {
  if (!roomId) return;
  // subscribe to room updates and merge into local turn state
}, [roomId]);

function playMove(move) {
  setState(s => applyMove(s, move));
  if (roomId) publishMove(roomId, move);
}
